# 建筑标准文件管理系统 (test-eBook)

一个基于现代化前后端分离架构的建筑标准文件（PDF）管理系统。提供标准文件的分类管理、安全上传、预览、下载以及核心特性的**自动化 OCR 识别与标准信息提取**。

---

## ✨ 核心特性

### 📂 大规模文件与分类管理
- **多层级智能分类树**：灵活自定义标准文件目录树，实现按细分业务场景组织。
- **文件全生命周期**：支持高速上传、属性编辑、无缝预览（内置 PDF.js）、授权下载。
- **文档自动化属性提取**：全自动识别并分离提取：标准号、名称及发布日期（Baidu/Paddle OCR 支持）。
- **软删除与回收站**：具备完整的回收站机制，支持文档还原与物理彻底删除。

### 👨‍💼 增强的管理后台
- **用户体系与权限**：
  - 支持 **Admin/User** 角色分配与细粒度权限控制。
  - 完备的 **用户生命周期管理**：支持管理员进行用户增删改查、角色切换及强制密码重置。
  - 开放 **用户注册功能**：支持外部用户注册并在登录后使用系统。
- **配置化面板**：具备站点动态配置注入功能，支持 OCR 引擎 (Baidu Cloud, PaddleOCR API) 在控制端进行热更新与连接测试。
- **多准则审计系统**：
  - 全链路操作审计：高度敏感操作（文件移除、属性修改、登录）强制历史记录溯源。
  - **Action 智能映射**：自动识别 UPLOAD/DELETE/EDIT/LOGIN 等核心动作，并支持前端多维查询过滤。

### 📊 运营可视化
- **仪表盘统计**：内置数据看板，支持文档总量、待处理 OCR 任务、活跃用户等核心指标实时汇总展现。

## 🛠 现代化技术选型

### 前端开发 (test-ebook-web)
- **核心构建**：Vue 3 + Vite + TypeScript (Node 20+ 构建环境)
- **基座周边**：Pinia (状态管理) + Vue Router
- **UI 生态系**：Element Plus 搭配深度自定义 Premium 主题风格
- **性能与体积**：PDF.js (内嵌预览)、Gzip 资源预压缩、Vendor 分包加载与资源拆分优化

### 后端开发 (test-ebook-api)
- **性能基座**：Go (1.21+) / Gin Web Framework (带有完备的优雅停机扩展)
- **数据库范式**：纯 Go 版 SQLite (`glebarez/sqlite`) + GORM（启用 WAL，优化并发上限）
- **安全加固**：JWT (含 bcrypt 盐值认证)、CORS 规范化拦截
- **工具链集成**：Zap 结构化日志、Viper 配置中心 (支持环境变量覆盖)

## 🚀 生产环境部署 (Docker)

本项目全架构经由 Docker 多阶段构建 (Multi-stage Build) 逻辑生成，部署过程轻量且环境一致。

### 1. 快速启动
确认服务器已具备 `Docker` 以及 `Docker Compose` 环境。在根目录下运行：

```bash
docker-compose up -d --build
```

### 2. 访问入口
- **前端门户**: `http://localhost:3000` (映射容器内 Nginx 80 端口)
- **后端 API**: `http://localhost:8182` (映射容器内 API 服务端口)

### 3. 数据层持久化规约
在启动后，将在根目录下挂载自动生成的核心数据存储卷：
- `backend-data:/app/data`：存储 SQLite 核心库 (`test_ebook.db`)。
- `backend-uploads:/app/uploads`：存放用户上传的所有原生标准 PDF 文件。

## 💻 本地开发协作指南

### 统一启动入口 (推荐)
已在项目根目录下通过 `concurrently` 建立了统一启动脚本，可一键联机：

```bash
# 1. 安装根目录管理依赖 (开发期)
npm install

# 2. 启动前后端服务 (全自动联调模式)
npm run dev
```

### 分模块手动启动
若需单独调试特定模块：
- **后端 API**: `npm run dev:api` (内部执行 `go run cmd/server/main.go`)
- **前端 Web**: `npm run dev:web` (内部执行 Vite 开发环境)
