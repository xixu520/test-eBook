# PLAN-26: 重构文档管理模块

## 目标
严格重构文档管理前端界面，杜绝 Bug，对齐后端 API，统一 UI 风格并应用真实的错误处理逻辑。

## 发现的问题
1. **ID 字段大小写不一致**：前端接口定义使用小写 `id`，而后端 `StandardFile` 模型使用大写 `ID`。这会导致删除、重试 OCR 等按 ID 操作的功能在实际测试时传入 `undefined` 或 0。
2. **编辑功能未完全实现**：`AdminDocumentsPage.vue` 中的 `submitEdit` 只有个假的功能（`ElMessage.success('修改成功（模拟）')`），且没有对接真实的后端修改接口。
3. **列表加载分类名**：表格目前只展示 `category_id` 而不是查字典对应出真实的分类名称。
4. **状态类型错误**：与 CategoryPage 同样的报错，例如 `:type="getStatusType(row.status)"` 如果返回不支持的类型（如 `''`），Vue 会抛出警告。
5. **UI 风格统一**：缺少与 CategoryPage 一致的高级感 (Premium) UI 处理，如统一的搜索占位、卡片样式增强、无数据时更优美的空状态等。

## 计划步骤（已完成）

- [x] 5. 自动化测试文档管理列表删除、表格解析显示结果。
