## 一、技术选型

| 层面 | 方案 | 说明 |
|------|------|------|
| 框架 | **Vue 3** + Composition API | 渐进式框架，生态成熟 |
| 构建工具 | **Vite 6** | 开发热更新快，生产打包高效 |
| 语言 | **TypeScript** | 类型安全，便于团队协作 |
| UI 组件库 | **Element Plus** | Vue 3 生态首选，控件丰富，主题可定制 |
| 路由 | **Vue Router 4** | 支持 `<keep-alive>` 页面缓存 |
| 状态管理 | **Pinia** | 官方推荐，简洁轻量 |
| HTTP 客户端 | **Axios** | 拦截器体系成熟 |
| PDF 预览 | **PDF.js (pdfjs-dist)** | API 明确要求 |
| 图标 | **@element-plus/icons-vue** + 自定义 SVG | 统一图标体系 |
| CSS 方案 | **CSS Variables** + **SCSS** + Element Plus 主题覆盖 | 支持浅/深色切换 |
| 字体 | 中文：霞鹜文楷(CDN)；英文：Inter(Google Fonts)；等宽：JetBrains Mono | 调查表确认 |
| 代码规范 | ESLint + Prettier + `<script setup>` | 统一格式 |

## 三、路由设计

| 路由路径 | 页面 | 权限 | 布局 |
|----------|------|------|------|
| `/login` | 登录页 | 公开 | 无布局（全屏居中卡片） |
| `/register` | 注册页 | 公开（受 `allow_registration` 控制） | 无布局（全屏居中卡片） |
| `/` | 首页-文件列表工作台 | 登录用户 | MainLayout |
| `/preview/:id` | PDF 预览页 | 登录用户 | PreviewLayout |
| `/admin` | 管理员仪表盘 | admin | AdminLayout |
| `/admin/categories` | 分类管理 | admin | AdminLayout |
| `/admin/upload` | 上传管理 | admin | AdminLayout |
| `/admin/recycle` | 回收站 | admin | AdminLayout |
| `/admin/audit-logs` | 审计日志 | admin | AdminLayout |
| `/admin/settings` | 系统设置 | admin | AdminLayout |
| `/admin/announcements` | 公告管理 | admin | AdminLayout |
| `/403` | 无权限 | 公开 | 空布局 |
| `/404` | 页面不存在 | 公开 | 空布局 |

### 路由守卫逻辑

1. **全局前置守卫**：检查 `token` 存在性 → 无 token 跳转 `/login`
2. **角色校验**：`/admin/*` 路由检查 `user.role === 'admin'` → 非管理员跳转 `/403`
3. **页面切换动画**：`<transition name="fade">` 实现淡入淡出（调查表确认）

| 文件列表 | `el-table` | 显示当前上传批次的文件及状态 |
| OCR 进度通知 | `ElNotification` + 自定义组件 | 右下角常驻，显示 task 轮询进度 |
| OCR 状态 | `el-tag` + `el-progress` | pending→排队中/processing→识别中 X%/completed→完成/failed→失败 |
| 重新 OCR | `el-button` size="small" | 失败时显示，调用 `POST /documents/{id}/ocr/retry` |
| OCR 结果预填 | `el-form` 自动填充 | completed 时自动填入 `standard_no`、`name`、`issue_date` |

**OCR 轮询逻辑**（`composables/useOcrPolling.ts`）：
- 间隔：每 3 秒调用 `GET /tasks/{task_id}/status`
- 上限：最多 100 次（约 5 分钟）
- 超时处理：展示"识别超时，请稍后重试"+ 重新 OCR 按钮

**API 映射**：
- 上传：`POST /api/v1/documents/upload` (FormData)
- 轮询：`GET /api/v1/tasks/{task_id}/status`
- 重新 OCR：`POST /api/v1/documents/{id}/ocr/retry`

## 十一、Docker 容器化部署方案

### 11.1 整体架构（精简双容器）

| 容器 | 基础镜像 | 职责 |
|------|---------|------|
| **frontend** | `nginx:1.27-alpine` | 托管 Vue 3 SPA 静态文件 + 反向代理 API 请求 |
| **backend** | `golang:1.23-alpine`（构建） → `alpine:3.20`（运行） | Go API 服务 + 内嵌 SQLite + 直接调用百度云 OCR API |

> **架构变更说明**：
> - ~~MySQL~~ → **SQLite**（嵌入式，零运维，数据文件挂载 Volume 持久化）
> - ~~Redis + Python OCR 服务~~ → **Go 后端直接 HTTP 调用百度云 OCR API**（无需消息队列）
> - 从 5 个容器精简为 **2 个容器**，极大降低部署复杂度

### 11.2 SQLite 写入并发处理（重要）

SQLite 为单写多读模型，需要在后端代码层面做好并发控制：

| 策略 | 说明 |
|------|------|
| **WAL 模式** | 启用 `PRAGMA journal_mode=WAL`，允许读写并发，写操作不阻塞读 |
| **单写连接** | Go 中使用 **两个连接池**：1 个只写连接（`SetMaxOpenConns(1)`）+ 1 个多读连接池 |
| **Busy Timeout** | 设置 `PRAGMA busy_timeout=5000`（5 秒），写冲突时自动重试而非立即报错 |
| **事务控制** | 写操作使用 `BEGIN IMMEDIATE` 事务，提前获取写锁避免死锁 |
| **连接参数** | DSN 添加 `?_journal_mode=WAL&_busy_timeout=5000&_synchronous=NORMAL&_foreign_keys=ON` |

```go
// 后端推荐的 SQLite 连接初始化伪代码
// 写连接（全局唯一）
writeDB, _ := sql.Open("sqlite3", "file:data/test_ebook.db?_journal_mode=WAL&_busy_timeout=5000&_synchronous=NORMAL&_foreign_keys=ON")
writeDB.SetMaxOpenConns(1)  // 关键：只允许一个写连接

// 读连接池
readDB, _ := sql.Open("sqlite3", "file:data/test_ebook.db?mode=ro&_journal_mode=WAL")
readDB.SetMaxOpenConns(4)   // 读可以并发
```

### 11.3 OCR 架构变更

原方案通过 Redis 消息队列分发给独立 Python 服务，现改为：

```
上传 PDF → Go 后端保存文件 → Go 启动 Goroutine
  → 提取 PDF 第一页图片（使用 Go PDF 库）
  → HTTP POST 调用百度云 OCR 通用文字识别 API
  → 解析返回结果，提取标准号/名称/日期
  → 更新 SQLite 数据库中的文档记录和任务状态
```

前端轮询逻辑不变（`GET /tasks/{task_id}/status`），后端任务状态直接写入 SQLite。

### 11.6 Nginx 配置 `nginx.conf`

```nginx
server {
    listen       8180;
    server_name  _;
    root         /usr/share/nginx/html;
    index        index.html;

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml;
    gzip_min_length 256;

    # 静态资源缓存（Vite 生成带 hash 文件名）
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # API 反向代理（转发到后端 Go 服务）
    location /api/ {
        proxy_pass http://backend:8181;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        # PDF 文件上传支持大体积
        client_max_body_size 60m;
    }

    # SPA 路由回退
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
}
```
