### 分层架构

```
HTTP 请求 → Router → Middleware → Handler → Service → Repository → SQLite
                                     ↓
                                 pkg (工具)
```

| 层 | 职责 | 规则 |
|----|------|------|
| **Handler** | 解析请求参数、调用 Service、返回 JSON | 不含业务逻辑 |
| **Service** | 业务逻辑编排、事务管理、权限校验 | 可调用多个 Repository |
| **Repository** | 纯数据库 CRUD 操作 | 不含业务逻辑，仅 SQL/GORM 调用 |
| **Model** | 数据库表映射 + DTO | GORM 标签定义表结构 |
| **Middleware** | 横切关注点 | 认证、日志、CORS、Recovery |
| **pkg** | 可复用工具 | JWT、哈希、OCR 封装 |

// 写连接（全局唯一，串行写入）

  "file:data/test_ebook.db?_journal_mode=WAL&_busy_timeout=5000&_synchronous=NORMAL&_foreign_keys=ON"

sqlDB, _ := writeDB.DB()
sqlDB.SetMaxOpenConns(1)       // 关键：单写
sqlDB.SetMaxIdleConns(1)

// 读连接池（并发安全）
readDB = gorm.Open(sqlite.Open(
  "file:data/test_ebook.db?mode=ro&_journal_mode=WAL"
), &gorm.Config{})
sqlDB2, _ := readDB.DB()

sqlDB2.SetMaxIdleConns(2)

## 五、中间件实现详解

### 5.1 JWT 认证中间件 (`middleware/auth.go`)

**流程**：
1. 从 Header 提取 `Authorization: Bearer <token>`
2. 解析 JWT，校验签名和过期时间
3. 从 payload 中提取 `user_id`，查询用户信息
4. 将 `user` 对象写入 `gin.Context` (`c.Set("user", user)`)
5. 失败返回 `401 { code: 401, message: "未登录或Token已失效" }`

**JWT Payload 结构**：
```json
{
  "user_id": 1,
  "username": "admin",
  "role": "admin",
  "exp": 1711108800    // 过期时间戳（24 小时）
}
```

**密钥来源**：`config.yaml` 中的 `jwt.secret` 或环境变量 `JWT_SECRET`

### 5.2 角色权限中间件 (`middleware/role.go`)

**流程**：
1. 从 `c.Get("user")` 获取用户
2. 检查 `user.Role == "admin"`
3. 非管理员返回 `403 { code: 403, message: "无权限" }`

### 5.3 请求日志中间件 (`middleware/logger.go`)

使用 Zap 记录：方法、路径、状态码、耗时、客户端 IP、User-Agent。

### 5.4 CORS 中间件 (`middleware/cors.go`)

允许前端开发环境跨域：`Access-Control-Allow-Origin: *`（生产环境由 Nginx 反代，不存在跨域）。

### 5.5 Panic 恢复 (`middleware/recovery.go`)

捕获 Panic，记录堆栈到 Zap，返回 `500 { code: 500, message: "服务器内部错误" }`。

## 六、各模块 Handler / Service 实现详解

### 6.1 认证模块

#### POST /auth/login — 用户登录

| 层 | 实现 |
|----|------|
| **Handler** | 解析 JSON `{username, password, captcha_code}` → 调用 `AuthService.Login()` → 返回 token + user |
| **Service** | ① `UserRepo.FindByUsername(username)` → ② `bcrypt.CompareHashAndPassword(hash, password)` → ③ 生成 JWT（24h 过期）→ ④ `AuditService.Log(LOGIN, userId, ip)` → ⑤ 返回 token + user DTO |
| **错误处理** | 用户名不存在/密码错误 → `400 "用户名或密码错误"`（不区分，防枚举） |

#### GET /auth/me — 获取当前用户

| 层 | 实现 |
|----|------|
| **Handler** | 从 `c.Get("user")` 取用户 → 返回 id/username/role/permissions |
| **权限字段** | `permissions` 存储为 JSON 字符串，查询时反序列化为数组 |

#### PUT /users/me/theme — 更新主题

| 层 | 实现 |
|----|------|
| **Handler** | 解析 `{theme}` → `UserRepo.UpdateTheme(userId, theme)` |
| **校验** | theme 必须是 `light` / `dark` 之一 |

#### POST /auth/register — 用户注册

| 层 | 实现 |
|----|------|
| **Handler** | 解析 JSON `{username, password}` → 调用 `AuthService.Register()` |
| **Service** | ① 查询 `SettingRepo.Get("allow_registration")` → 若 `"false"` 返回 `403 "系统当前未开放注册"` → ② 校验 `username` 长度 ≥ 3 、`password` 长度 ≥ 6 → ③ `UserRepo.FindByUsername(username)` 检查重复 → 已存在返回 `400 "用户名已被占用"` → ④ `bcrypt.GenerateFromPassword(password, 10)` → ⑤ `UserRepo.Create({username, password_hash, role: "user", permissions: ["download"]})` → ⑥ 审计日志 `LOGIN` → ⑦ 返回 `{id, username, role}` |
| **错误处理** | 注册关闭→403；用户名重复→400；参数不合法→400 |

### 6.2 分类管理模块

#### GET /categories — 获取分类列表

| 层 | 实现 |
|----|------|
| **Handler** | 调用 `CategoryService.GetTree()` → 返回平铺列表 |
| **Service** | `CategoryRepo.FindAll()` 取全部 → 用 SQL 子查询统计每个分类的 `doc_count`：`SELECT c.*, (SELECT COUNT(*) FROM documents WHERE category_id = c.id AND is_deleted = 0) AS doc_count FROM categories c ORDER BY sort_order` |
| **说明** | 前端 UI 调查表选择不显示 doc_count，但 API 仍返回以保留灵活性 |

#### POST /categories — 新增分类

| 层 | 实现 |
|----|------|
| **Handler** | 解析 `{parent_id, name, sort_order}` → `CategoryService.Create()` |
| **Service** | ① 校验 `parent_id` 存在（0 除外） → ② 校验同级下名称不重复 → ③ `CategoryRepo.Create()` |
| **审计** | 记录 `EDIT` 日志 |

#### PUT /categories/:id — 修改分类

| 层 | 实现 |
|----|------|
| **Service** | ① 查找原分类 → ② **根节点降级保护**：若原 `parent_id=0` 且新 `parent_id≠0`，拒绝（`400 "禁止将顶级分类移动至其他分类下"`）→ ③ 更新 |

#### DELETE /categories/:id — 删除分类

| 层 | 实现 |
|----|------|
| **Service** | ① **子分类检查**：`CategoryRepo.CountChildren(id)` > 0 → `400 "该分类下存在子分类"` → ② **文件关联检查**：`DocumentRepo.CountByCategory(id)` > 0 → `400 "该分类下仍有标准文件"` → ③ `CategoryRepo.Delete(id)` |

### 6.3 标准文件模块

#### GET /documents — 文件列表查询

| 层 | 实现 |
|----|------|
| **Handler** | 解析 Query: `page, size, keyword, category_id` → `DocumentService.List()` |
| **Service** | GORM 动态 Where 链式查询 |
| **SQL 逻辑** | `WHERE is_deleted = 0 AND (standard_no LIKE '%keyword%' OR name LIKE '%keyword%') AND category_id = ?` → 联表查 `categories.name AS category_name` → 分页 `LIMIT size OFFSET (page-1)*size` |
| **角色差异** | Service 层判断 `user.Role`：admin 返回全部字段；user 排除 `uploader_name, upload_time, view_count, download_count, active_version` |
| **搜索高亮** | 后端不处理高亮，前端自行用正则包裹 `<mark>` |

#### GET /documents/:id/preview — PDF 预览

| 层 | 实现 |
|----|------|
| **Handler** | ① `DocumentRepo.FindById(id)` → ② 读取 `file_path` 文件 → ③ 设置 `Content-Type: application/pdf` → ④ `c.File(filePath)` 流式返回 → ⑤ 异步 `DocumentRepo.IncrViewCount(id)` (Goroutine 更新，避免阻塞响应) |
| **浏览计数** | 使用 Goroutine + writeDB 执行 `UPDATE documents SET view_count = view_count + 1 WHERE id = ?` |

#### GET /documents/:id/download — 文件下载

| 层 | 实现 |
|----|------|
| **Handler** | ① 权限检查 `user.Permissions` 含 `"download"` → ② `c.FileAttachment(filePath, fileName)` → ③ 异步 `IncrDownloadCount(id)` |
| **响应头** | `Content-Disposition: attachment; filename="GB50007-2011.pdf"` |

#### POST /documents/upload — 上传文件

| 层 | 实现 |
|----|------|
| **Handler** | ① `c.FormFile("file")` 获取文件 → ② 检查大小 ≤ 50MB → ③ 调用 `DocumentService.Upload()` |
| **Service 详细流程** | |

```
1. 读取文件流，计算 SHA256 哈希
2. 查询 DocumentRepo.FindByHash(hash)
   ├─ 命中且文件存在 → 秒传：复用记录，返回 {is_duplicate: true, document_id}
   └─ 未命中 → 继续
3. 生成存储路径：uploads/{YYYY-MM}/{uuid}.pdf
4. 保存文件到磁盘
5. 创建 Document 记录（ocr_status='pending'）
6. 创建 OcrTask 记录（task_id=UUID, status='pending'）
7. 启动 Goroutine → OcrService.Process(task_id, filePath)

```

| **审计** | 记录 `UPLOAD` 日志 |

### 6.4 OCR 与核验模块

#### OCR 处理服务 (`service/ocr_service.go`)

**完整流程（Goroutine 内执行）**：

```
func (s *OcrService) Process(taskId, filePath string) {
  1. 更新任务状态 → processing, progress=10

  2. 使用 pdfcpu 提取 PDF 第一页为 PNG 图片
     → 临时文件 /tmp/{taskId}_page1.png
     → progress=30

  3. 读取图片，Base64 编码

  4. HTTP POST 百度云 OCR API
     URL: https://aip.baidubce.com/rest/2.0/ocr/v1/accurate_basic
     Headers: Content-Type: application/x-www-form-urlencoded
     Body: image=<base64>&access_token=<token>
     → progress=70

  5. 解析 OCR 返回结果 words_result[]
     → 用正则匹配标准号：/(GB|GB\/T|JGJ|CJJ|...)\s*\d+[-—]\d{4}/
     → 提取名称：标准号后的连续文字行
     → 提取日期：匹配 /\d{4}[-年]\d{1,2}[-月]\d{1,2}/
     → progress=90

  6. 更新 OcrTask：status=completed, progress=100,
     result_standard_no, result_name, result_issue_date

  7. 自动填充 Document 的 standard_no, name, issue_date
     （仅当字段为空时填充，不覆盖已有值）

  8. 更新 Document：ocr_status='completed'

  9. 清理临时文件
}
```

**百度云 OCR Access Token 管理**（`pkg/baidu_ocr.go`）：

```
- 使用 API Key + Secret Key 获取 access_token
  POST https://aip.baidubce.com/oauth/2.0/token
    ?grant_type=client_credentials
    &client_id={API_KEY}
    &client_secret={SECRET_KEY}
- Token 有效期 30 天，内存缓存 + 过期前自动刷新
- 使用 sync.RWMutex 保护并发访问
```

**错误处理**：
- PDF 解析失败 → `status=failed, error="PDF 文件解析失败"`
- 百度 API 超时/错误 → `status=failed, error="OCR 服务调用失败: <detail>"`
- 正则未匹配到标准号 → `status=completed`（标记完成但结果字段为空，前端可手填）

#### GET /tasks/:task_id/status — 轮询 OCR 进度

## 十、安全措施

| 措施 | 实现 |
|------|------|
| 密码存储 | bcrypt (cost=10)，永不存储明文 |
| JWT 密钥 | 至少 32 字节随机字符串，通过环境变量注入 |
| SQL 注入 | GORM 参数化查询，无原生拼接 SQL |
| 文件上传校验 | ① 检查 MIME 为 `application/pdf` ② 检查文件头魔术字节 `%PDF` ③ 大小限制 50MB |
| 路径穿越 | 文件存储使用 UUID 命名，禁止用户控制文件路径 |
| 权限隔离 | Handler → Service 层按角色过滤数据 |
| CORS | 生产环境由 Nginx 处理，后端仅开发环境允许 |
| 速率限制 | 登录接口限制 5次/分钟（使用内存计数器） |
